This week, I wanted to get some more information on Meta Viewport. A good portion of this
came from a project in Highschool. My group made a computer game and in our code we used
a lot of fixed sizes (10 px, 100 px, etc.). This worked perfectly on the screens that we
developed on, but when I tried to play the game on my 4K laptop, the whole game was around
3 inches wide. The meta viewport tag is mainly used to make websites developed for computers
easily readable on mobile devices. By now, many mobile browsers support the meta tag. In the tag
that I used on this webpage, you can see that the width is set to device-width. This sets the width
to the width of the device that you're viewing the site from. This isn't dependent on how large
or small the screen is. The other portion is initial scale being set to 1. This just changes
the zoom of the page.

I also learned that you can set the width field of the meta tag to a specific width in order to cater
to smaller screens. This width field, when given a specific number, acts as a minimum width. If this width
is set to 300, then 300px is the minimum width it can be. If the screen is larger than 300 pixels, the browser
chooses to expand the size of the viewport, as opposed to zooming in. There are other field that can be added
such as maximum and minimum scale and user-scalable. The article I read mentioned that its common for web
developers for mobile devices to set a maximum scale as too much zoom can have negatives effects on the page.
